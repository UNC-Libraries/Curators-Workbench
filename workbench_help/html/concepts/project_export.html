Export and Import of Projects and Files

Once arrangement and description has been completed to your satisfaction, you may wish to export the project manifest. You may wish to export for submission into a repository. Alternately, you may want a compact, machine-readable inventory of the collection contents with a record of preliminary authentication, descriptive, and preservation activities performed. Currently, the workbench supports one export format: submission to the Carolina Digital Repository. The export format is a METS XML file, with MODS elements included from a crosswalk or other user-added description.

Other forms of export are also available from the Export Menu. These include:

  • Export Arrangement to CSV file
  • Export (and Import) of project to ZIP archive
  • Export of Arrangement to BagIt ZIP Package
  • Export (and Import) of Crosswalks, Dictionaries, Forms and other individual project files

Project export/import to ZIP can be used to share work between users. For instructions, see Sharing Projects via Import and Export.

Additional project exports are possible to support work flow needs. Share your needs on GitHub or via the users forum.
